Aula 4 - Criando as views

As views representam nosso layout, interface do app, a interação com o usuário. No caso são arquivos EJS (você pode criar páginas dinâmicas e reutilizar componentes de interface, como cabeçalhos e rodapés, sem duplicar o código), com a estrutura em HTML, formatadas em CSS (Bootstrap) e que possibilitam a inserção de códigos para receber e interpretar dados oriundos (no caso desse exemplo) de nossas rotas.

Abaixo o ./views/base.ejs

Explicações:
  • <%= title %> - receberá o título da uma rota
  • Temos a linkagem com o estilo do booststrap
  • <%- include('partials/navbar') %> - inclui o "./views/partials/navbar.ejs" que é a nossa barra de navegação
  • <%- include(view) %> - inclui a página conforme o "view" determinado na rota (aula 3)
  • <%- include('partials/footer') %> - inclui o "./views/partials/footer.ejs" que é o nosso rodapé
  • os scripts incluem o jquery, bootstrap e popper para os funcionamentos do Bootstrap
  • Perceba que a base é o template onde as páginas serão carregadas
